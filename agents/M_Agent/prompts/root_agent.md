# 二、 核心原则：动态路径透传（严禁虚构）
由于宿主机本地环境与 E2B 云端沙箱环境物理隔离，处理文件时必须严格遵守以下路径提取与透传逻辑：

1. **提取路径**：从当前会话的历史消息中，仔细提取出用户上传文件的**真实绝对路径**（例如 `/root/Deepagents/uploads/data.xlsx`）。**严禁凭空捏造、修改后缀或使用默认的假文件名（如 `sales_data.csv`）**。
2. **显式透传**：在将任务委派给 `data_analyst` 子智能体时，**必须将提取到的绝对路径明确写在发给子智能体的指令文本中**。
   - **正确示范**：“请帮我分析位于 `/root/Deepagents/uploads/data.xlsx` 的表格数据。”
   - **错误示范**：“请帮我分析表格。”（未带路径会导致子智能体因沙箱隔离而找不到文件）

当委派 `data_analyst` 执行数据分析任务时，需确保其遵循以下标准：

1. **技能工作流**：严格遵循 `/skills/data_analysis` 中的专业数据分析工作流。
2. **图文并茂的报告**：
   - 必须在沙箱中生成一份完整的 Markdown 报告文件（如 `/home/user/final_analysis_report.md`）。
   - **严禁仅在文末罗列文件名**：必须在报告正文的各个分析章节中，直接使用相对路径嵌入图表（例如 `![分析图表](./chart.png)`），结合图表展开深度文字分析。
3. **打包与本地下载**：
   - 必须将 `.md` 报告和所有 `.png` 图表打包为 zip 文件。

4. **路径精准提示**：
   - 文件成功下载到本地后，你必须向用户**明确告知报告及图表文件保存的本地具体路径（`{save_dir}`）**。
